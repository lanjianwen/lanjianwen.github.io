Java集合

为什么要使用Java集合

数组弊端:
1.数组一旦初始化，长度就已经不可再改变
2.数组声明的类型初始化就决定数组类型，不可更改
3.数组数据有序，可重复

Collection父接口，对象的集合，单列集合

常用方法

  1. add(Object obj)添加元素
  2. addAll(Collection coll)添加一个集合
  3. size()集合大小
  4. isEmpty()判断集合是否为空
  5. clear()清空集合
  6. contains(Object obj)判断是否包含某元素，通过元素的equals方法来判断
  7. containsAll(Collection coll)判断是否包含某集合，通过元素的equals方法来判断
  8. remove(Object obj)移除某元素
  9. removeAll(Collection coll)移除集合中所有包含coll集合
  10. retainsAll(Collection coll)返回两个集合的交集
  11. equals(Object obj)比较两个对象是否相等
  12. hasCode()获取哈希值
  13. toArray()集合转换为数组
  14. iterator()返回迭代器对象，用于集合遍历

List接口，有序，可重复的集合

  • 常用方法
  1. void add(int index, Object ele):在index位置插入ele元素
  2. boolean addAll(int index, Collection eles):从index位置开始将eles中的所有元素添加进来
  3. Object get(int index):获取指定index位置的元素
  4. int indexOf(Object obj):返回obj在集合中首次出现的位置
  5. int lastIndexOf(Object obj):返回obj在当前集合中末次出现的位置
  6. Object remove(int index):移除指定index位置的元素，并返回此元素
  7. Object set(int index, Object ele):设置指定index位置的元素为ele
  8. List subList(int fromIndex, int toIndex):返回从fromIndex到toIndex位置的子集合
    ArratList接口实现类，数组存储，没有同步，线程不安全,效率高
  • 底层原理
  1. JDK1.7：ArrayList像饿汉式，直接创建一个初始容量为10的数组
  2. JDK1.8：ArrayList像懒汉式，一开始创建一个长度为0的数组，当添加第一个元素时再创建一个始容量为10的数组
  3. Arrays.asList(…) 方法返回的 List 集合，既不是ArrayList 实例，也不是Vector 实例。Arrays.asList(…) 返回值是一个固定长度的 List 集合
    LinkedList接口实现类，链表存储，没有同步，线程不安全
  • 新增方法
  1. void addFirst(Object obj)开头添加一个元素
  2. void addLast(Object obj)末尾添加一个元素
  3. Object getFirst()获取首个元素
  4. Object getLast()获取最后一个元素
  5. Object removeFirst()移除首个元素
  6. Object removeLast()移除末尾元素
    Vector 接口实现类 数组存储，同步，线程安全
  7. void addElement(Object obj) 添加元素
  8. void insertElementAt(Object obj,int index) 指定索引添加元素
  9. void setElementAt(Object obj,int index) 指定索引修改元素
  10. void removeElement(Object obj) 移除元素
  11. void removeAllElements() 移除所有元素

    Set接口，无序，不可重复的集合

    HashSet接口实现类，哈希表存储
  • 添加元素过程
  1. 当向 HashSet 集合中存入一个元素时，HashSet 会调用该对象的 hashCode() 方法来得到该对象的 hashCode 值，然后根据 hashCode 值，通过某种散列函数决定该对象在 HashSet 底层数组中的存储位置。
  2. 如果两个元素的hashCode()值相等，会再继续调用equals方法，如果equals方法结果为true，添加失败；如果为false，那么会保存该元素，但是该数组的位置已经有元素了，那么会通过链表的方式继续链接。
  3. 如果两个元素的 equals() 方法返回 true，但它们的 hashCode() 返回值不相等，hashSet 将会把它们存储在不同的位置，但依然可以添加成功。
    LinkedHashSet继承HashSet链表存储
  4. LinkedHashSet 根据元素的 hashCode 值来决定元素的存储位置，但它同时使用双向链表维护元素的次序，这使得元素看起来是以插入顺序保存的。
  5. LinkedHashSet插入性能略低于 HashSet，但在迭代访问 Set 里的全部元素时有很好的性能
  6. LinkedHashSet 不允许集合元素重复
    TreeSet实现类，底层为二叉树
  7. TreeSet 是 SortedSet 接口的实现类，TreeSet 可以确保集合元素处于排序状态。
  8. TreeSet 两种排序方法：自然排序和定制排序。默认情况下，TreeSet 采用自然排序。
  9. TreeSet底层使用红黑树结构存储数据

    Map接口，键值对(key,value)的集合,双列集合

    Map用于保存具有映射关系的数据，Map里保存着两组数据：key和value，它们都可以使任何引用类型的数据，但key不能重复(使用set)。所以通过指定的key就可以取出对应的value,可以重复(使用collection)
  • 常用方法
  1. Object put(Object key,Object value)：将指定key-value添加到(或修改)当前map对象中
  2. void putAll(Map m):将m中的所有key-value对存放到当前map中
  3. Object remove(Object key)：移除指定key的key-value对，并返回value
  4. void clear()：清空当前map中的所有数据
  5. Object get(Object key)：获取指定key对应的value
  6. boolean containsKey(Object key)：是否包含指定的key
  7. boolean containsValue(Object value)：是否包含指定的value
  8. int size()：返回map中key-value对的个数
  9. boolean isEmpty()：判断当前map是否为空
  10. boolean equals(Object obj)：判断当前map和参数对象obj是否相等
  11. Set keySet()：返回所有key构成的Set集合
  12. Collection values()：返回所有value构成的Collection集合
  13. Set entrySet()：返回所有key-value对构成的Set集合

    HashMap实现类，没有同步，线程不安全

  14. 允许使用null键和null值，与HashSet一样，不保证映射的顺序。
  15. 所有的key构成的集合是Set:无序的、不可重复的。所以，key所在的类要重写：equals()和hashCode()
  16. 所有的value构成的集合是Collection:无序的、可以重复的。所以，value所在的类要重写：equals()
  17. 一个key-value构成一个entry
  18. 所有的entry构成的集合是Set:无序的、不可重复的
  19. HashMap 判断两个 key 相等的标准是：两个 key 通过 equals() 方法返回 true， hashCode 值也相等。
  20. HashMap 判断两个 value相等的标准是：两个 value 通过 equals() 方法返回 true。
  • 存储
  1. JDK 7及以前版本：HashMap是数组+链表结构(即为链地址法)
  2. JDK 8版本发布以后：HashMap是数组+链表+红黑树实现。
    LinkdeHashMap 继承HashMap，双向链表存储
  3. LinkedHashMap 是 HashMap 的子类
  4. 在HashMap存储结构的基础上，使用了一对双向链表来记录添加元素的顺序
  5. 与LinkedHashSet类似，LinkedHashMap 可以维护 Map 的迭代顺序：迭代顺序与 Key-Value 对的插入顺序一致

    Hashtables实现类，同步，线程安全

  6. Hashtable是个古老的 Map 实现类，JDK1.0就提供了。不同于HashMap， Hashtable是线程安全的。
  7. Hashtable实现原理和HashMap相同，功能相同。底层都使用哈希表结构，查询速度快，很多情况下可以互用。
  8. 与HashMap不同，Hashtable 不允许使用 null 作为 key 和 value
  9. 与HashMap一样，Hashtable 也不能保证其中 Key-Value 对的顺序
  10. Hashtable判断两个key相等、两个value相等的标准，与HashMap一致。

    TreeMap 旧版jdk实现类，同步，线程安全

  11. TreeMap存储 Key-Value 对时，需要根据 key-value 对进行排序。 TreeMap 可以保证所有的 Key-Value 对处于有序状态。
  12. TreeSet底层使用红黑树结构存储数据
  13. TreeMap 的 Key 的排序：
  14. 1 自然排序：TreeMap 的所有的 Key 必须实现 Comparable 接口，而且所有的 Key 应该是同一个类的对象，否则将会抛出 ClasssCastException
  15. 2 定制排序：创建 TreeMap 时，传入一个 Comparator 对象，该对象负责对TreeMap 中的所有 key 进行排序。此时不需要 Map 的 Key 实现Comparable 接口
  16. TreeMap判断两个key相等的标准：两个key通过compareTo()方法或者compare()方法返回0。
    Properties用来处理配置文件，key和value都是String类型
  17. Properties 类是 Hashtable 的子类，该对象用于处理属性文件
  18. 由于属性文件里的 key、value 都是字符串类型，所以 Properties 里的 key和 value 都是字符串类型
  19. 存取数据时，建议使用setProperty(String key,String value)方法和getProperty(String key)方法
